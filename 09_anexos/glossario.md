# Glossário

## A

**AUC-ROC:** Area Under the ROC Curve. Métrica de performance que varia de 0.5 (aleatório) a 1.0 (perfeito).

**Average Precision (AP):** Métrica que resume a curva precision-recall.

## C

**CNPJ:** Cadastro Nacional da Pessoa Jurídica. Identificador único de empresas no Brasil.

**CNPJ Básico:** 8 primeiros dígitos do CNPJ, identificando a empresa matriz.

**CNAE:** Classificação Nacional de Atividades Econômicas. Código que identifica o setor de atuação.

**CatBoost:** Categorical Boosting. Algoritmo de gradient boosting otimizado para dados categóricos.

**Chunk:** Pedaço de dados processado incrementalmente para economizar memória.

## E

**Encoding:** Conversão de variáveis categóricas em numéricas para uso em modelos.

**Ensemble:** Combinação de múltiplos modelos para melhor predição.

**Engagement Rate:** Taxa de engajamento = (curtidas / seguidores) × 100.

## F

**Feature:** Variável preditiva usada pelo modelo de Machine Learning.

**Feature Engineering:** Processo de criar novas features a partir de dados brutos.

**F1-Score:** Média harmônica de precision e recall.

## G

**Gradient Boosting:** Técnica de ensemble que treina modelos sequencialmente, cada um corrigindo erros do anterior.

## H

**Hiperparâmetro:** Parâmetro do modelo definido antes do treino (ex: max_depth, learning_rate).

**Hybrid Strategy:** Estratégia que gera múltiplos datasets (unified + separate).

## I

**Imputação:** Preenchimento de valores ausentes (missing values).

**Imputer:** Ferramenta que realiza imputação (ex: SimpleImputer).

## L

**LightGBM:** Light Gradient Boosting Machine. Variante rápida de gradient boosting.

**Liability of Newness:** Teoria que empresas jovens têm maior risco de falência.

## M

**MAR:** Missing At Random. Padrão de dados ausentes.

**MEI:** Microempreendedor Individual.

**MNAR:** Missing Not At Random. Dados ausentes com padrão sistemático.

## O

**Optuna:** Framework de otimização de hiperparâmetros.

**Overfitting:** Modelo aprende ruído do treino, não generaliza para teste.

## P

**Pipeline:** Sequência de etapas de processamento de dados até modelo final.

**Porte:** Classificação de empresa por tamanho (MEI, Micro, Pequena, Média, Grande).

**Precision:** Proporção de predições positivas que estão corretas.

## R

**Random Forest:** Ensemble de múltiplas árvores de decisão.

**Recall:** Proporção de casos positivos corretamente identificados.

**Regularização:** Técnica para prevenir overfitting (ex: L1, L2).

**ROC Curve:** Receiver Operating Characteristic. Gráfico de TPR vs FPR.

## S

**Scaler:** Ferramenta para normalizar features (ex: StandardScaler).

**SHAP:** SHapley Additive exPlanations. Técnica de explicabilidade de modelos.

**Sobrevivência:** No contexto deste projeto, empresa que permaneceu aberta durante evento crítico.

**Split:** Divisão de dados em treino, validação e teste.

**Stratified Split:** Split que mantém proporção de classes.

## T

**Target:** Variável a ser predita (ex: sobreviveu_pandemia).

**TPE:** Tree-structured Parzen Estimator. Algoritmo de otimização Bayesiana.

**Tree-based:** Modelos baseados em árvores de decisão.

**Trial:** Uma tentativa de treinamento com hiperparâmetros específicos no Optuna.

## X

**XGBoost:** Extreme Gradient Boosting. Implementação otimizada de gradient boosting.
